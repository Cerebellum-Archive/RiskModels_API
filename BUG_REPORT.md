# RiskModels API Bug Report (v3.0.0-agent)

This report summarizes critical, high, and low severity bugs identified during the systematic QA scan of the RiskModels API.

## Critical Severity

| ID | Endpoint | Description | Expected | Actual |
|---|---|---|---|---|
| BUG-001 | `/api/metrics/{ticker}` | **Broken Authentication (BOLA/Missing Auth)**. The endpoint is accessible without a Bearer token or with an invalid/random token. | 401 Unauthorized | 200 OK |
| BUG-002 | `/api/metrics/{ticker}` | **Authentication Bypass**. Endpoint does not validate the Bearer token against any backend store; any string is accepted. | 401 Unauthorized | 200 OK |

## High Severity

| ID | Endpoint | Description | Expected | Actual |
|---|---|---|---|---|
| BUG-003 | `/api/metrics/{ticker}` | **Data Integrity Failure**. All risk metrics (`l3_market_er`, etc.) are returning `null` or `0.0000` for active tickers (e.g., NVDA). | Valid numeric risk data | All fields `null` |
| BUG-004 | `/api/auth/token` | **Auth Flow Failure**. OAuth2 token exchange returns 401 for validly formatted client credentials. | 200 OK with JWT | 401 Unauthorized |

## Low Severity

| ID | Endpoint | Description | Expected | Actual |
|---|---|---|---|---|
| BUG-005 | `/api/batch/analyze` | **Inconsistent Status Code**. Invalid tickers return `status: "error"` with an error message instead of the documented `not_found`. | `status: "not_found"` | `status: "error"` |
| BUG-006 | Various | **Missing Rate Limit Headers**. Metered endpoints are not returning `X-RateLimit-Remaining` or `X-RateLimit-Limit` headers. | Numeric headers | `null` |
